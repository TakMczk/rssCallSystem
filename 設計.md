# 詳細設計書（RSS技術記事スコアリング & 再配信システム）

## 1. 全体構成

- 実行方式: スケジュールバッチ（GitHub Actions Cron）
- リポジトリ構造（案）:

```
/ (root)
  要件定義.md
  設計.md
  README.md
  pyproject.toml or requirements.txt
  /src
    __init__.py
    config.py
    main.py              # エントリポイント（バッチ）
    fetcher.py           # RSS 取得 & 正規化
    parser_utils.py      # パース補助（抜粋生成など）
    scorer.py            # LLM 呼出 & スコアリング
    ranking.py           # ソートロジック
    rss_builder.py       # 出力 RSS 生成
    models.py            # データモデル (pydantic など)
    logging_utils.py     # ログ設定
    retry.py             # 共通リトライユーティリティ
  /tests

    test_models.py
    test_validation.py
  /docs
    rss.xml              # 生成成果物（GitHub Pages 公開）
```

## 2. 技術選定詳細

| 項目 | 選択 | 理由 |
|------|------|------|
| 言語 | Python 3.11+ | 型ヒント / ライブラリ豊富 / 起動軽量 |
| 依存管理 | uv or pip | シンプル運用 |
| RSS パース | feedparser | 実績豊富・緩い RSS/Atom 差異吸収 |
| HTTP | httpx | 非同期 + タイムアウト制御容易 |
| モデル定義 | pydantic | バリデーション容易 |
| LLM API | openai | 指定要件 |
| テスト | pytest | 標準的 |
| Lint/Format | ruff + black | 速度 / 一貫性 |
| ログ | 標準 logging (構造化は将来) | シンプル |

## 3. コンポーネント設計

### 3.1 config.py

- 環境変数 / 定数集中管理
- 内容:

```
FEED_URLS: list[str]
TOP_N: int = 30
REQUEST_TIMEOUT: float = 15.0
FETCH_CONCURRENCY: int = 5
SCORE_MODEL: str (例: "gpt-4o-mini")
OPENAI_API_KEY: str (env)
RETRY_MAX: int (fetch用)
LLM_TEMPERATURE: float = 0.1
MAX_SCORE_RETRY: int = 2
LOG_LEVEL: str
```

### 3.2 models.py

- Pydantic モデル
  - RawFeedItem (パース直後)
  - Article (正規化後, スコア前)
  - ScoreResult (LLM 出力)
  - RankedArticle (スコア付与後)

### 3.3 fetcher.py

- 非同期 fetch(url) → bytes
- parse_feed(url) → list[RawFeedItem]
- normalize(raw) → Article
- 重複排除: set で id 集約
- 並列処理: asyncio.gather + セマフォ

### 3.4 parser_utils.py

- excerpt 生成（description / content から N 文字抽出）
- HTML タグ除去 (bleach / 自前正規表現)

### 3.5 scorer.py

- build_prompt(article)
- call_openai(prompt) with retry (429/5xx)
- parse_score(json_str) → ScoreResult
- fail-safe: フォールバック (5 点x3)
- バッチ化検討: 初期は1 item 1 call

### 3.6 ranking.py

- sort_articles(articles: list[RankedArticle]) -> list[RankedArticle]
- キー: (-total, -novelty, -expertise, -interest, -published_at)

### 3.7 rss_builder.py

- build_rss(articles: list[RankedArticle]) -> str(XML)
- feed metadata 設定
- description にスコア内訳 + reason
- Python: ElementTree or feedgen 利用（軽量性から手書きも可）

### 3.8 logging_utils.py

- get_logger(name)
- JSON 形式は将来、初期は: `%(asctime)s %(levelname)s %(name)s: %(message)s`

### 3.9 retry.py

- 汎用 async retry(func, exceptions, tries, backoff, jitter)

### 3.10 main.py

フロー:

```
async def run():
  load config
  fetch & normalize all feeds
  score sequential or limited parallel
  rank
  rss_xml = build_rss(top_n)
  write docs/rss.xml (ensure dir)
```

同期エントリ: `if __name__ == "__main__": asyncio.run(run())`

## 4. データフロー

1. FEED_URLS を iteration → fetch → parse → RawFeedItem
2. RawFeedItem -> Article (normalize + id 生成)
3. Article -> ScoreResult (LLM)
4. Article + ScoreResult -> RankedArticle
5. sort -> 上位 TOP_N -> RSS 生成

## 5. エラーハンドリング詳細

| 層 | 例外 | 対応 |
|----|------|------|
| fetch | httpx.TimeoutException | リトライ → ログ WARN → スキップ |
| parse | ValueError (必須欠落) | スキップ |
| scoring | openai.error.RateLimitError | バックオフ再試行 |
| scoring | JSONDecodeError | 再試行 (MAX_SCORE_RETRY) → フォールバック |
| output | IOError | 再試行 / 失敗でプロセス exit(1) |

## 6. 並列戦略

- フェッチ: 5 並列 (FEED_URL 数少ないためオーバーヘッド低) Item 数多時はまとめ
- スコアリング: 3 並列程度（レート調整）→ セマフォ

## 7. セキュリティ

- API Key: `OPENAI_API_KEY` 参照のみ
- レスポンス & ログに Key 出力禁止

## 8. パフォーマンス最適化余地

- スコアリング前 Title + Summary だけ使用 → Token 削減
- まとめスコア（N 件 JSON 配列）→ JSON 出力整形
- キャッシュ: 同リンク・同タイトル短期間再出現時は前回スコア再利用（KV / ローカル JSON）

## 13. Pydantic モデル案

```
class RawFeedItem(BaseModel):
  source: str
  title: str
  link: HttpUrl | str
  published: datetime | None
  summary: str | None
  content: str | None

class Article(BaseModel):
  id: str
  source: str
  title: str
  url: HttpUrl | str
  published_at: datetime
  summary: str
  excerpt: str

class ScoreResult(BaseModel):
  novelty: int
  interest: int
  expertise: int
  reason: str
  @property
  def total(self):
    return self.novelty + self.interest + self.expertise

class RankedArticle(Article):
  scores: ScoreResult
```
